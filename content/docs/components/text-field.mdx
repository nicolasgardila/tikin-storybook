## Instalacion

## Uso

### Tamaños

El TextField viene en 4 tamaños con diferentes alturas y radios.

| Tamaño | Altura | Radius | Padding | Font size |
|--------|--------|--------|---------|-----------|
| xs | 32px | 6px | 4px 8px | 12px |
| sm | 40px | 8px | 8px 12px | 14px |
| md | 48px | 12px | 12px 16px | 16px |
| lg | 56px | 12px | 14px 16px | 16px |

## Anatomia

Estructura del TextField con floating label (tamaño md):

```
┌─────────────────────────────────────┐
│  padding: 0 16px                    │
│                                     │
│  Label (Ubuntu Sans, 10px, 0.5 op)  │
│  Value (Ubuntu Sans, 14px)          │
│                                     │
│         border: 0.5px #DEDEDE       │
│         border-radius: 12px         │
│         height: 48px                │
│         bg: white                   │
└─────────────────────────────────────┘
gap: 8px (si hay error)
Feedback (Ubuntu Sans, 13px, #EC221F)
```

| Parte | Token | Valor |
|-------|-------|-------|
| Border (default) | — | 0.5px solid #DEDEDE |
| Border (active) | — | 0.5px solid #0B0B0B + ring 1px |
| Border (error) | — | 1px solid #EC221F |
| Border radius (md) | `--ds-radius-lg` | 12px |
| Height (md) | `--ds-height-md` | 48px |
| Padding (md) | `--ds-space-4` | 0 16px |
| Label font | Ubuntu Sans | 10px / 400 |
| Value font | Ubuntu Sans | 14px / 400 |
| Label default font | Funnel Display | 16px / 400 |
| Error bg | — | #FEE9E7 |
| Error color | — | #EC221F |
| Feedback font | Ubuntu Sans | 13px / 400 |

## Guías de Uso

<DoDont type="do">
  El label vive dentro del campo y actúa como placeholder en estado default. Al activar el campo, el label se reduce y sube para dar espacio al valor.
</DoDont>

<DoDont type="dont">
  No uses un label externo separado del input. El patrón floating label mantiene la asociación visual directa entre label y valor.
</DoDont>

<DoDont type="do">
  Muestra mensajes de error específicos debajo del campo con el color de error (#EC221F) indicando cómo corregir el problema.
</DoDont>

<DoDont type="dont">
  No uses mensajes genéricos como "Campo inválido". Indica claramente qué se espera (ej: "Ingresa un correo válido").
</DoDont>
