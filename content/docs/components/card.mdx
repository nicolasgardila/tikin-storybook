## Uso

```tsx title="card.tsx"
import { Card } from '@dstikin/ds'

export default function Example() {
  return (
    <Card variant="default" padding="default">
      <h3>Cuenta de Ahorro</h3>
      <p>$24,850.00</p>
    </Card>
  )
}
```

### Interacciones

El componente incluye feedback tactil al presionar: escala sutil (`scale 0.97`) con reduccion de opacidad, y un release con transicion spring natural. Este efecto se desactiva automaticamente en estados `disabled` y `loading`.

## Especificaciones

| Propiedad | Valor |
|-----------|-------|
| Border radius | 24px |
| Border width | 0.5px solid |
| Warm border | `#E8E4DD` (dark: `#2A2826`) |
| Neutral border | `#DEDEDE` (dark: `#333333`) |
| Padding default | 20px |
| Padding compact | 16px |
| Gap default | 16px |
| Gap compact | 12px |

## Anatomia

Estructura de la Card (variante default warm):

```
┌─────────────────────────────────────┐
│  border: 0.5px #E8E4DD (warm)       │
│  border-radius: 24px                │
│  bg: #FFFFFF                         │
│                                      │
│  ┌─ Card Body ──────────────────┐   │
│  │  padding: 20px                │   │
│  │  gap: 16px                    │   │
│  │                               │   │
│  │  Title (Funnel Display)       │   │
│  │  Subtitle (Ubuntu Sans)       │   │
│  │  Content area                 │   │
│  │                               │   │
│  └───────────────────────────────┘   │
│                                      │
│  Compact variant: padding 16px,      │
│  gap 12px                            │
└──────────────────────────────────────┘
```

| Parte | Token | Valor |
|-------|-------|-------|
| Border | — | 0.5px solid |
| Warm border | `--ds-warm-border` | #E8E4DD |
| Neutral border | `--ds-neutral-border` | #DEDEDE |
| Border radius | `--ds-radius-2xl` | 24px |
| Body padding | `--ds-space-5` | 20px |
| Body gap | `--ds-space-4` | 16px |
| Compact padding | `--ds-space-4` | 16px |
| Compact gap | `--ds-space-3` | 12px |

## Guías de Uso

<DoDont type="do">
  Usa cards warm (default) para contenido financiero, wallet y cuentas. El tono cálido comunica confianza y pertenencia.
</DoDont>

<DoDont type="dont">
  No mezcles cards warm y neutral en el mismo contenedor. Mantén consistencia de tono dentro de cada sección.
</DoDont>

<DoDont type="do">
  Usa la variante elevated para cards que flotan sobre el contenido, como actividad reciente o notificaciones rápidas.
</DoDont>

<DoDont type="dont">
  No combines elevated con outline. Elige un solo método de separación visual: sombra o borde, no ambos.
</DoDont>
