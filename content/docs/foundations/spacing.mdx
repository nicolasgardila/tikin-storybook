## Escala de Espaciado

El sistema de espaciado de Black Cash esta basado en una unidad de **4px**. Todos los tokens son multiplos de esta unidad base, creando ritmo visual consistente en toda la interfaz.

---

## Uso en Codigo

```tsx
{/* Padding de tarjeta con gap interno */}
<div
  className="p-[var(--ds-space-6)]"
  style={{ gap: 'var(--ds-space-3)' }}
>
  <h3>Titulo</h3>
  <p>Contenido</p>
</div>

{/* Espaciado entre secciones */}
<section className="mb-[var(--ds-space-8)]">
  <div className="space-y-[var(--ds-space-4)]">
    <input className="p-[var(--ds-space-3)]" />
    <input className="p-[var(--ds-space-3)]" />
  </div>
</section>
```

## Principios

### Cuando usar cada nivel

| Nivel | Rango | Contexto |
|-------|-------|----------|
| Micro | 0-2 (0-8px) | Dentro de componentes: icon gaps, inline spacing |
| Interno | 3-4 (12-16px) | Padding interno de componentes: inputs, chips, tags |
| Componente | 5-6 (20-24px) | Padding de tarjetas, dialogos, paneles |
| Seccion | 8-12 (32-48px) | Separacion entre secciones y grupos |
| Layout | 14-16 (56-64px) | Margenes de pagina, separacion mayor |

### Regla de consistencia

Usar siempre tokens del sistema en lugar de valores arbitrarios. Si un espaciado no encaja en ningun token, elegir el mas cercano — la consistencia visual es mas importante que la precision al pixel.
