<task type="auto">
  <name>Task 1: Apply DS typography to content area</name>
  <files>app/global.css</files>
  <action>
    1. Add typography CSS overrides to `app/global.css` AFTER the existing token blocks. These override fumadocs-ui's default typography with DS fonts:

       ```css
       /* ==========================================================================
          Content Area Typography — apply DS fonts to documentation content
          Overrides fumadocs-ui defaults to use production DS typography.
          ========================================================================== */

       /* Headings use Funnel Display */
       .prose h1,
       .prose h2,
       .prose h3,
       .prose h4,
       .prose h5,
       .prose h6 {
         font-family: var(--font-funnel-display), system-ui, sans-serif;
       }

       /* Body text uses Ubuntu Sans */
       .prose {
         font-family: var(--font-ubuntu-sans), system-ui, sans-serif;
       }

       /* Code uses Ubuntu Sans Mono */
       .prose code,
       .prose pre code {
         font-family: var(--font-ubuntu-sans-mono), ui-monospace, monospace;
       }

       /* Page title (DocsTitle) uses Funnel Display */
       [data-docs-page-title],
       .fd-docs-title {
         font-family: var(--font-funnel-display), system-ui, sans-serif;
       }
       ```

       IMPORTANT:
       - Use the CSS variable names from next/font (`--font-funnel-display`, `--font-ubuntu-sans`, `--font-ubuntu-sans-mono`) — NOT the Tailwind utility names.
       - Target `.prose` which is the fumadocs content area class.
       - If fumadocs-ui uses different class names for the content area, inspect the rendered HTML and adjust selectors accordingly. The pattern `[data-*]` or `.fd-*` prefixes are common in fumadocs-ui.
       - Do NOT remove any existing CSS. Append these overrides at the end of the file.

    2. Verify fonts apply correctly by checking:
       - Headings (h1-h6) show Funnel Display
       - Paragraphs show Ubuntu Sans
       - Code blocks show Ubuntu Sans Mono
       - If fonts don't apply, the CSS variable names from next/font might differ. Check `lib/fonts.ts` for the exact `variable` values.
  </action>
  <verify>
    - `npm run build` succeeds
    - No CSS compilation errors
    - The CSS file contains font overrides targeting .prose elements
  </verify>
  <done>DS typography applied to content area — Funnel Display headings, Ubuntu Sans body, Ubuntu Sans Mono code</done>
