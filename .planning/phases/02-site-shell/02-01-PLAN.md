    1. Update `app/docs/layout.tsx` to configure sidebar with tabs and header nav:

       ```tsx
       import { DocsLayout } from 'fumadocs-ui/layouts/docs';
       import type { ReactNode } from 'react';
       import { source } from '@/lib/source';

       export default function Layout({ children }: { children: ReactNode }) {
         return (
           <DocsLayout
             tree={source.pageTree}
             nav={{
               title: 'Black Cash DS',
             }}
             sidebar={{
               defaultOpenLevel: 1,
             }}
           >
             {children}
           </DocsLayout>
         );
       }
       ```

       Key configuration:
       - `nav.title`: Site branding in header. Keep it simple — "Black Cash DS".
       - `sidebar.defaultOpenLevel: 1`: Categories (Foundations, Components, Guides) are visible with first level expanded.
       - Do NOT use sidebar tabs — the folder structure with meta.json already creates the category grouping in the sidebar. Tabs are for multiple documentation sets (like API vs Guides), not for subcategories within one set.

    2. Verify the fumadocs-ui DocsLayout already provides:
       - Dark/light mode toggle button in the header (via RootProvider theme)
       - Responsive sidebar that collapses to a sheet/drawer on mobile
       - Search button in header (will be wired in Phase 4)

       If dark mode toggle is NOT visible, check that `RootProvider` in app/layout.tsx does not have `disableThemeToggle` set. The toggle should appear automatically.

    IMPORTANT: Do NOT over-customize the layout. fumadocs-ui already matches the shadcn/ui docs pattern closely. The default DocsLayout provides the sidebar + content layout that SHELL-01 requires. Focus on configuration, not custom components.
