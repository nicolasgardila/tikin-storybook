  <name>Task 1: Set up fumadocs search with static search index</name>
  <files>app/api/search/route.ts, app/layout.tsx, app/docs/layout.tsx</files>
  <action>
    fumadocs-ui includes a built-in search dialog that opens with Cmd+K. It needs a search API to index content.

    1. Check what fumadocs-ui already provides out of the box:
       - The `RootProvider` may already render a search button in the header
       - The `DocsLayout` may already have search integration
       - Check if Cmd+K already triggers anything without additional setup

    2. Create the search API route. fumadocs provides a static search API that indexes MDX content at build time:

       Create `app/api/search/route.ts`:
       ```typescript
       import { source } from '@/lib/source';
       import { createSearchAPI } from 'fumadocs-core/search/server';

       export const { GET } = createSearchAPI('advanced', {
         indexes: source.getPages().map((page) => ({
           title: page.data.title,
           description: page.data.description,
           url: page.url,
           id: page.url,
           structuredData: page.data.structuredData,
         })),
       });
       ```

       IMPORTANT: The exact API may differ in fumadocs-core v16. Check the imports:
       - `createSearchAPI` may be from `fumadocs-core/search/server`
       - The `structuredData` field is generated by fumadocs-mdx for heading-level search
       - If `createSearchAPI` doesn't exist, check for `createFromSource` or similar utility

    3. If fumadocs-ui's search dialog doesn't connect automatically to this API route, configure it:
       - Check if `RootProvider` needs a `search` prop
       - The search may use `useDocsSearch` hook with `type: 'static'` pointing to the API route
       - fumadocs-ui may handle this automatically if the API route is at `/api/search`

    4. Verify Cmd+K opens the search dialog and returns results.

    IMPORTANT: fumadocs-ui v16 likely handles most of this automatically. Read the actual fumadocs source/types before creating files. The search button should already be in the header via DocsLayout. You may only need the API route.
  </action>

    - Typing "buttons" returns the Buttons component page
    - Typing "colores" returns the Colors foundations page

  </verify>
