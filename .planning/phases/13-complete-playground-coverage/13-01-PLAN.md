## Tasks

<task id="1" type="auto">
**Create DS components for Container, Badge, and Status Chips**

Create three DS component files following the ds-button.tsx pattern:

1. **components/ds/ds-container.tsx** — DSContainer
   - Props: variant (double/simple/warm), padding (default/compact)
   - Renders a container shell with variant-specific backgrounds and borders
   - Uses --preview-* CSS vars for theme adaptation
   - Default content showing a sample card or content block inside

2. **components/ds/ds-badge.tsx** — DSBadge
   - Props: variant (count/dot), count (number), max (number), showZero (boolean)
   - Renders a badge indicator (count number or dot) on a sample icon
   - Brand bg #0B0B0B, Ubuntu Sans Mono 500 10px, min-width 16px r8

3. **components/ds/ds-status-chips.tsx** — DSStatusChips
   - Props: variant (success/warning/error/info/neutral/brand), label (text), size (xs/sm/md)
   - Renders a status tag pill with variant-specific colors
   - Ubuntu Mono 12px, r44, letter-spacing 0.6px, border 0.5px

<done>
- All three DS component files exist and export their components
- Each component renders with inline styles and --preview-* CSS vars
- Components accept the specified props with sensible defaults
</done>
</task>

<task id="2" type="auto">
**Create remaining DS components + all playground wrappers + register + add to MDX**

1. Create remaining DS components:

   **components/ds/ds-dropdown.tsx** — DSDropdown
   - Props: state (default/active), label (text)
   - Renders a dropdown in open state showing menu items
   - Double-layer container pattern, items 10px 16px padding

   **components/ds/ds-radio-button.tsx** — DSRadioButton
   - Props: checked (boolean), disabled (boolean), label (text)
   - Renders a radio button with optional label
   - Funnel Display font for label, filled circle when selected

2. Create playground wrappers for all 5 components:

   **container-playground.tsx** controls:
   - variant: select, default 'double', options ['double', 'simple', 'warm']
   - padding: select, default 'default', options ['default', 'compact']

   **badge-playground.tsx** controls:
   - variant: select, default 'count', options ['count', 'dot']
   - count: number, default 3, min 0, max 99
   - max: number, default 99, min 1, max 999
   - showZero: boolean, default false

   **status-chips-playground.tsx** controls:
   - variant: select, default 'success', options ['success', 'warning', 'error', 'info', 'neutral', 'brand']
   - label: text, default 'Activo'
   - size: select, default 'sm', options ['xs', 'sm', 'md']

   **dropdown-playground.tsx** controls:
   - state: select, default 'active', options ['default', 'active']
   - label: text, default 'Opciones'

   **radio-button-playground.tsx** controls:
   - checked: boolean, default false
   - disabled: boolean, default false
   - label: text, default 'Opcion 1'

3. Register all 5 playground components in mdx-components.tsx

4. Add playground as first element after intro in each MDX page:
   - container.mdx: `<ContainerPlayground />`
   - badge.mdx: `<BadgePlayground />`
   - status-chips.mdx: `<StatusChipsPlayground />`
   - dropdown.mdx: `<DropdownPlayground />`
   - radio-button.mdx: `<RadioButtonPlayground />`

</task>
