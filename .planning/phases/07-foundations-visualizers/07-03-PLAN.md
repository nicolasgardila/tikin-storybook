    Create `components/docs/grid-visualizer.tsx` as a client component that visualizes the grid system.

    The component should:
    1. Be a `'use client'` component (for breakpoint selection)
    2. Accept no props — hardcode the DS grid system data
    3. Show a breakpoint selector: Desktop (12 cols), Tablet (8 cols), Mobile (4 cols)
    4. Render a visual grid container showing:
       - Column bars with semi-transparent color (e.g., brand color at 10% opacity)
       - Gutter spaces between columns clearly visible
       - Column numbers labeled (1, 2, 3... up to N)
       - Container margins on left/right
    5. Below the column overlay, show the grid specs for the selected breakpoint:
       - Columns count
       - Gutter width
       - Margin width
       - Max width
    6. Optionally show an example layout overlay (e.g., 8+4 split on desktop, full width on mobile)

    Grid data:
    ```ts
    const breakpoints = [
      {
        name: 'Desktop',
        label: '> 1024px',
        columns: 12,
        gutter: 16,
        margin: 24,
        maxWidth: 1280,
      },
      {
        name: 'Tablet',
        label: '768–1024px',
        columns: 8,
        gutter: 16,
        margin: 20,
        maxWidth: null, // 100%
      },
      {
        name: 'Mobile',
        label: '< 768px',
        columns: 4,
        gutter: 12,
        margin: 16,
        maxWidth: null, // 100%
      },
    ];
    ```

    Layout implementation:
    - The container has a fixed height (~200px) with a subtle background
    - Render N equal-width columns using CSS grid or flexbox
    - Each column is a colored bar (use --ds-brand at ~10% opacity, or a light blue)
    - Gutters are the gaps between columns
    - Margins are padding on the container sides
    - Label each column with its number centered at the top

    Breakpoint selector: Use the same pill-button pattern as ComponentPreview toolbar (segmented control with active state).
