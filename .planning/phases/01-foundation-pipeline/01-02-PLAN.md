         subsets: ['latin'],

         variable: '--font-ubuntu-sans',
         display: 'swap',

         subsets: ['latin'],
         variable: '--font-ubuntu-sans-mono',
         display: 'swap',
         weight: ['400', '500', '600', '700'],
       });

<task type="auto">
  <name>Task 2: Set up namespaced DS tokens and docs theme in Tailwind v4</name>
  <files>app/global.css</files>
  <action>
    1. Update `app/global.css` with Tailwind v4 `@theme` configuration. Structure:

       **Fumadocs imports** (keep existing):
       ```css
       @import 'tailwindcss';
       @import 'fumadocs-ui/css/neutral.css';
       @import 'fumadocs-ui/css/preset.css';
       ```

       **DS Font families** (map CSS variables from next/font to Tailwind):
       ```css
       @theme {
         --font-ds-display: var(--font-funnel-display), system-ui, sans-serif;
         --font-ds-body: var(--font-ubuntu-sans), system-ui, sans-serif;
         --font-ds-mono: var(--font-ubuntu-sans-mono), ui-monospace, monospace;
       }
       ```

       **DS Color tokens** — namespaced with `--ds-*`, defined as CSS custom properties in `:root` and `.dark`:
       ```css
       :root {
         /* DS Warm tone (panels, cards, wallet, page bg) */
         --ds-warm-bg: #FAF8F4;
         --ds-warm-surface: #F1EFEB;
         --ds-warm-border: #E8E4DD;

         /* DS Neutral tone (dialog, tabs, inputs) */
         --ds-neutral-bg: #F6F6F6;
         --ds-neutral-surface: #FFFFFF;
         --ds-neutral-border: #DEDEDE;

         /* DS Brand */
         --ds-brand: #0B0B0B;
         --ds-brand-on: #FFFFFF;

         /* DS Spacing (subset — full list in Phase 6) */
         --ds-space-0: 0px;
         --ds-space-1: 4px;
         --ds-space-2: 8px;
         --ds-space-3: 12px;
         --ds-space-4: 16px;
         --ds-space-5: 20px;
         --ds-space-6: 24px;
         --ds-space-8: 32px;
         --ds-space-10: 40px;
         --ds-space-12: 48px;
         --ds-space-16: 64px;

         /* DS Radius */
         --ds-radius-sm: 8px;
         --ds-radius-md: 12px;
         --ds-radius-lg: 16px;
         --ds-radius-xl: 20px;
         --ds-radius-2xl: 24px;
         --ds-radius-pill: 44px;
       }

       .dark {
         /* DS Brand inversion for dark mode */
         --ds-brand: #FFFFFF;
         --ds-brand-on: #0B0B0B;

         /* DS tones stay same in dark (they represent component tokens, not site theme) */
       }
       ```

       **Docs site chrome tokens** — separate namespace for the documentation site itself:
       ```css
       :root {
         --docs-bg: #FFFFFF;
         --docs-fg: #0B0B0B;
         --docs-muted: #6B7280;
         --docs-border: #E5E7EB;
         --docs-accent: #0B0B0B;
       }

       .dark {
         --docs-bg: #0B0B0B;
         --docs-fg: #F9FAFB;
         --docs-muted: #9CA3AF;
         --docs-border: #374151;
         --docs-accent: #F9FAFB;
       }
       ```

    2. Do NOT create a `tailwind.config.ts` file — Tailwind v4 uses CSS-only configuration via `@theme`. Only create `tailwind.config.ts` if fumadocs requires it for content paths (check fumadocs docs). If needed, keep it minimal:
       ```typescript
       import type { Config } from 'tailwindcss';

       export default {
         content: [
           './content/**/*.{md,mdx}',
           './app/**/*.{ts,tsx}',
           './lib/**/*.{ts,tsx}',
           './mdx-components.tsx',
           './node_modules/fumadocs-ui/dist/**/*.js',
         ],
       } satisfies Config;
       ```

    IMPORTANT: The fumadocs CSS imports (`neutral.css`, `preset.css`) handle the base fumadocs theming. Our `--ds-*` tokens overlay on top, keeping DS values separate from docs chrome values. This separation is critical for later phases where component previews show DS tokens while the docs site uses its own theme.
  </action>

    - Dark mode toggle (fumadocs built-in) switches `.dark` class and `--docs-*` values change
    - Font families are accessible via Tailwind: `font-ds-display`, `font-ds-body`, `font-ds-mono`
    - `npm run build` succeeds
  </verify>
  <done>

  </done>
