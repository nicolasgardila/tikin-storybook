# Requirements: Black Cash Design System — Documentation Site

**Defined:** 2026-03-20
**Core Value:** Every component is documented with exact production tokens, real usage examples, and copy-paste code — so the team never has to guess how to use the DS.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Public-facing marketing site | Internal tool only |
| Component library npm package | Documentation only, not a distributable package |
| Figma plugin integration | Docs reference Figma but don't sync |
| Multi-language (i18n) | Spanish-only content is fine |
| Multi-framework examples | Team uses one stack (React + Tailwind) |
| WYSIWYG documentation editor | MDX is human-readable; teach basics instead |
| Component usage analytics | Small internal team; use Slack feedback |
| Figma live embeds | Slow, require auth, break on reorganization |
